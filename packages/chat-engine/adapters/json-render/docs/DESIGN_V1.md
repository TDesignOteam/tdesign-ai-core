## 1. 核心理念

### 1.1 设计哲学

基于 **生成式 UI** 模式的动态渲染方案，采用 **预定义 + 动态组合** 的安全架构：

```
AI 生成 Schema → 预定义组件渲染 → 预定义 Action 处理
```

**核心原则**：
- **安全性**：AI 只能使用预定义的组件和操作
- **可控性**：前端预先知道所有可能的 action
- **零侵入**：完全基于现有 Activity 机制扩展
- **高性能**：基于 JSON Patch 的增量渲染

### 1.2 技术选型

| 维度 | 选择 | 理由 |
|------|------|------|
| **核心框架** | `@json-render/react` | Vercel Labs 维护，生态完善 |
| **协议标准** | JSON Pointer + JSON Patch | RFC 标准，跨平台兼容 |
| **类型安全** | Zod Schema | 编译时 + 运行时双重保障 |
| **性能优化** | React.memo + 增量渲染 | 流式场景下的高频更新优化 |

#### 流式增量处理支持

##### A2UI 模式的流式处理
```
初始快照：createSurface → 完整 Surface State
     ↓
增量更新：updateComponents → 累积到现有 State  
     ↓
数据更新：updateDataModel → 更新 dataModel
     ↓
转换渲染：完整 State → json-render Schema → React 组件
```

##### 纯 json-render 模式的流式处理
**重要更正**：`@json-render/react` 的 `useUIStream` 并非原生流式支持，而是基于 JSON Patch 的增量更新：

```
AG-UI SSE Chunks → event-mapper 解析 → Activity Event → ActivityRenderer
                                                              ↓
                      json-render Schema → @json-render/react 引擎
```

**实际流式处理机制**：
- **AG-UI 协议层**：提供真正的流式数据传输和增量更新
- **useUIStream**：仅处理 JSON Patch 格式的增量更新，不是独立的流式方案
- **json-render 模式**：依然需要 AG-UI 协议提供流式能力

**关键差异**：
- **A2UI 模式**：需要自定义 buildSurfaceState 进行消息累积
- **json-render 模式**：直接使用 json-render Schema，但流式能力依然来自 AG-UI 协议层
- **共同点**：两种模式都依赖 AG-UI 协议提供流式传输和增量更新能力

```
┌─────────────────────────────────────────────────────────┐

│  ┌─────────────┐ ┌─────────────┐ ┌─────────────────┐   │
│  │扁平邻接表   │ │JSON Pointer │ │流式增量更新     │   │

│  └─────────────┘ └─────────────┘ └─────────────────┘   │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│                协议映射关系                              │
│  A2UI                    →    json-render               │
│  ┌─────────────────────────────────────────────────┐   │
│  │ id, component        →    key, type             │   │
│  │ 属性扁平化           →    props 对象包裹         │   │
│  │ child/children       →    children 数组         │   │
```

#### 转换复杂度分析

| 特性 | A2UI | json-render | 转换复杂度 |
|------|------|-------------|-----------|

### 4.2 A2UI v0.9 消息处理策略

#### 累积式状态构建

```
┌─────────────────────────────────────────────────────────┐
│               A2UI Messages 处理流程                     │
│                                                         │
│  createSurface    →  初始化 Surface 基础结构             │
│        ↓                                                │
│  updateComponents →  累积组件（数组形式）                │
│        ↓                                                │
│  updateDataModel  →  累积数据更新                        │
│        ↓                                                │
│  完整 Surface State                                      │
└─────────────────────────────────────────────────────────┘
```

#### 核心处理逻辑

```
buildSurfaceState(messages[]) {
  surfaceState = {}
  
  for each message in messages:
    if (createSurface)    → 初始化结构
    if (updateComponents) → 累积组件更新
    if (updateDataModel)  → 应用数据变更
  
  return 完整的 Surface State
}
```

## 5. 性能优化

### 5.1 渲染性能优化体系

#### 多层次优化策略

```
┌─────────────────────────────────────────────────────────┐
│                 性能优化金字塔                           │
│                                                         │
│  ┌─────────────────────────────────────────────────┐   │

│  └─────────────────────────────────────────────────┘   │
│              ┌─────────────────────────────┐           │
│              │        React 层优化          │           │
│              │ React.memo + 深比较 + 缓存   │           │
│              └─────────────────────────────┘           │
```

#### 关键优化技术

| 优化层级 | 技术手段 | 性能收益 |
|----------|----------|----------|
| **架构层** | 增量渲染 + deltaInfo | 减少 70% 渲染时间 |
| **React 层** | React.memo + react-fast-compare | 避免 90% 无效重渲染 |
| **组件层** | withA2UIBinding + 延迟解析 | 减少 50% 计算开销 |
